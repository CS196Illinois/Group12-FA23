How it Works

Introduction

The website takes input from the user (two countries, who has home field advantage, and the year). Then it uses a ML model to predict the probability of each side winning a soccer match (for countries).

Machine Learning

Data Cleaning

1. Imported the dataset from Kaggle
2. Removed any special characters, such as Ã , and replaced them with their standard character, in this case it would be a.
3. Split the data by the delimiters, and then assigned the splits to their appropriate data variable for their respective country
Variables we used and how they were cleaned:
- Year
- homeTeam - Team with home field advantage (must be a valid country otherwise the row is discarded)
- awayTeam - Team without home field advantage (must be a valid country otherwise the row is discarded)
- homeScore / awayScore - score of the match, if illegal values were detected (negative numbers or non-integers, the row was discarded)
- Neutral - Boolean value representing if the match was played on a neutral field, assumed to be negative for most of the data
- ***tourName, City, and Country variables were dropped as irrelevant variables from the uncleaned data .csv file

Testing

First round:
- Inputted only the columns containing the data for the following:home team name, away team name, home field advantage, date, and result.
- We asked the algorithm to predict the result.
- On average, the returned predictions had barely over 50% correctly categorized for 10-fold cross validation.
Second round:
- Filtered out smaller teams
- Inputted new data into the algorithm
- New model yields about 70% accuracy at predicting win/lose (nearly 0% predicting ties)
